## 2026-06-27 — EPG Guide Full Overhaul (5 Critical Bugs Fixed)

- **[Critical Bugfix]** `server.js`: Fixed `ReferenceError: exactMap is not defined` crashing every EPG lookup in both `/api/custom-epg/bulk` and `/api/custom-epg` routes. `exactMap` was referenced but never defined anywhere. Replaced with a proper 3-tier lookup: (1) `fuzzyMatchCache` (manually curated `epg_mappings.json`), (2) direct `epg_channel_id` match, (3) Jaccard bigram fuzzy match using the existing `jaccardSim` function. This was the showstopper causing zero programs to ever load.
- **[Critical Bugfix]** `server.js`: Added the missing `/api/epg/stream` SSE (Server-Sent Events) endpoint. The frontend was connecting to it on every startup but it didn't exist, causing the `EventSource` to retry with a 404 every 3 seconds — a continuous flood of HTTP requests saturating the server. The endpoint now holds the connection open, sends a keepalive heartbeat every 25s, and broadcasts `epg_updated` when the background EPG fetch completes.
- **[Critical Bugfix]** `server.js`: Fixed EPG boot-load cache reader using the wrong data structure. The old reader tried to iterate `parsed.programs` (a flat array) to rebuild `programsByChannel`, but `parsed.programs` didn't exist in the cache file — only `programsByChannel` (keyed object) was stored. Boot now always loaded zero programs even with a valid 273MB cache on disk. Also unified the cache write format to only store `programsByChannel` (dropped the redundant flat `programs` array), cutting cache file size roughly in half.
- **[Performance]** `EPGGrid.jsx`: Eliminated 50+ live Dexie IndexedDB subscriptions (one `useLiveQuery` per visible row). Lifted to a single parent-level `useLiveQuery` that fetches all programs for all channel IDs in one query via `anyOf()`, builds a keyed `Map<channelId, program[]>` in a single `useMemo`, and passes pre-sorted data to each `Row` via props. Rows now do an O(1) `Map.get()` lookup — no async DB calls. This is the root cause of the scrolling lag during EPG import.
- **[Bugfix]** `useAppStore.js`: Fixed timestamp type mismatch. Server sends `start_timestamp` as a string (e.g. `"1750000000"` in seconds). The old code did `string * 1000` which produces `NaN`. Fixed to `parseInt(..., 10)` first, with a guard to detect if the value is already in millisecond scale (> 1e11) to avoid double-multiplying.
- **[Maintenance]** Deleted stale 273MB `epg_cache.json` in old format. Server will regenerate it in the new unified `programsByChannel` format on next boot.
- **[Maintenance]** Removed duplicate `setTimeout(() => fetchAndParseCustomEpg(), 15000)` — replaced by the SSE-aware version near `app.listen` that fetches and broadcasts `epg_updated` upon completion.


- **[Bugfix]** Restored instant category switching in the Live TV EPG. The full channel catalog is now pre-loaded into the store at startup (`fetchInitialCategories`) and category filtering happens as a pure synchronous `useMemo` computation — no more async network re-fetch on every category tab click. Also fixed the category filter which was checking for `'all'` instead of `'0'` as the "All Channels" sentinel value.
- **[Bugfix]** Fixed the EPG grid rendering only the timeline bar with no channels, icons, or program rows underneath. The `category_id` field was being dropped from channel objects during store mapping, causing the UI's category filter to silently eliminate every channel from the list.
- **[Bugfix]** Fixed "Maximum update depth exceeded" React infinite render loop. `updateActiveProfile` in `useUserData.js` was creating new object references on every call even when no data changed, causing `LayoutManager.jsx` to re-trigger the category auto-hide effect infinitely. Added an object identity bailout to short-circuit no-op updates.
- **[Bugfix]** Fixed profile being lost on every app restart. Identified that the infinite render loop was also corrupting and resetting the `activeProfileId`, forcing the user to recreate their profile each session.
